## [2026-02-27] - feature/#188-status-ui-KSY

### Added & Changed (추가 및 변경 사항)

**1. 데이터 로딩 상태 (Skeleton UI) 구성**
  * **해상도별 공용 스켈레톤 컴포넌트 제작:** 데이터를 서버에서 불러오는 대기 시간 동안 빈 화면 대신 표출될 뼈대(Skeleton) UI를 디자인했습니다. 
  * 데스크톱/태블릿 공용(`❖ tbl-row-skeleton`) 및 모바일 V2용(`❖ m-tbl-row-skeleton`) 컴포넌트를 분리하여 제작했습니다.
  * 기존 텍스트, 순위 숫자, 로고, 게이지 바 등을 지정된 스켈레톤 배경색(#21242C) 도형으로 교체하여, 로딩 완료 후 실제 데이터 표출 시 화면이 흔들리는 레이아웃 시프트(CLS) 현상을 방지했습니다.

**2. 데이터 없음 (Empty State) UI 구성**
  * **상태 메시지(`Status_Message`) 컨테이너 설계:** 검색 결과가 없거나 아직 리스트가 비어있을 때 사용자가 겪을 혼란을 방지하기 위한 화면 중앙 안내 영역을 구축했습니다.
  * 데스크톱의 넓은 여백부터 좁은 모바일 화면까지 시각적 밸런스가 유지되도록 크기를 최적화했으며, Secondary 컬러(#9194A1)의 텅 빈 박스 아이콘과 "조회된 종목이 없습니다." 문구를 배치하여 직관성을 높였습니다.

**3. 에러 (Error State) UI 구성 및 액션 유도**
  * **예외 상황 시각화:** 서버 통신 실패 등으로 데이터를 불러오지 못했을 때를 대비한 에러 화면을 데스크톱, 태블릿, 모바일에 각각 적용했습니다.
  * 경고 아이콘에 Error 상태 컬러(#EC1A13)를 적용하고, "데이터를 불러오는 데 실패했습니다."라는 명확한 문구를 제공했습니다.
  * **'다시 시도' 버튼 도입:** 사용자가 에러 화면(Dead-end)에 갇히지 않도록 Primary 보라색(#6D4AE6)을 활용한 강조 버튼을 추가하여, 즉각적인 문제 해결 액션을 유도하는 UX를 완성했습니다.
