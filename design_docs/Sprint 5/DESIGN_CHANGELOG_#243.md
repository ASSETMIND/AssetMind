## [2026-03-14] - feature/#243-ui-states-KSY

### Added & Changed (추가 및 변경 사항)

**1. 데이터 로딩 상태 (Skeleton UI) 설계**
* **영역별 스켈레톤 적용:** 차트, 호가창, 거래현황 등 주요 데이터 패널의 레이아웃 형태를 유지한 Skeleton UI를 제작하여, 데이터 로딩 중 발생할 수 있는 시각적 이질감을 최소화했습니다.
* **디자인 시스템 통일:** AssetMind 다크 테마에 맞춘 단색 톤 블록과 일관된 모서리 둥글기를 적용하여 전체적인 UI 통일성을 부여했습니다.

**2. 예외 상황 (Error & Empty State) UI 설계**
* **오류 (Local Error) UI 구성:** 특정 API 통신 실패 시 화면 전체가 아닌 해당 데이터 영역에만 제한적으로 노출되는 부분 오류 컴포넌트를 설계하여 사용자의 인지적 혼란을 방지했습니다.
* **명확한 피드백 및 복구 액션 제공:** 데이터 미수신 시 직관적인 경고 아이콘과 구체적인 안내 텍스트, 그리고 '다시 시도' 버튼을 배치하여 사용자가 즉각적으로 상황을 파악하고 대처할 수 있도록 구성했습니다.
* **시장 휴장 (Empty State) 대응:** 장 운영 외 시간에 빈 화면이 노출되지 않도록, 차트 및 호가 영역에 휴장 상태를 알리는 명확하고 친절한 형태의 Empty State UI를 적용했습니다.
