## 1. 개요 (Overview)

본 문서는 AssetMind 로그인 프로세스의 핵심 비즈니스 로직을 담당하는 커스텀 훅 `useLoginLogic`의 동작을 검증하기 위한 단위 테스트(Unit Test) 명세이다.
폼 데이터 제출(Submit) 시 발생하는 API 통신, 보안 토큰(Access Token) 처리, 전역 상태 동기화, 그리고 HTTP 상태 코드별 에러 핸들링이 의도대로 동작하는지 확인한다.

### 1.1. 테스트 환경

- **Framework:** Jest, `@testing-library/react` (React Hooks Testing Library)
- **Mocking Strategy:**
  - `useLogin` (React Query Mutation) 격리
  - `useAuthStore` (Zustand Global State) 격리
  - `react-router-dom`, `axios` 유틸리티 격리
- **Key Verification:**
  - **Success Flow:** 토큰 저장(Memory), 스토어 업데이트, 페이지 이동 연계 확인
  - **Error Handling:** 401/404(인증 실패)와 기타 에러(500 등)의 UI 피드백 분기 검증
  - **UX/UI Logic:** 성공 시에만 토스트 노출, 실패 시 Input Field 에러 바인딩 확인
