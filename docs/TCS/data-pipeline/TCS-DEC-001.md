# Logging Decorator Module

## 1. 개요 (Overview)

- **대상 모듈:** `log_decorator.py`
- **작성 목적:** 동기/비동기 함수 실행 시 로깅, 컨텍스트 관리, 예외 처리, 데이터 마스킹(PII) 기능이 명세대로 정확히 동작하는지 검증한다.
- **테스트 범위:** `LoggingDecorator` 클래스와 내부 헬퍼 함수(`_sanitize_args`, `_truncate_output`)의 모든 분기.

## 2. 테스트 환경 및 전략 (Test Environment & Strategy)

- **Unit Test Framework:** `pytest`를 사용하며, 비동기 테스트를 위해 `pytest-asyncio`를 활용한다.
- **Mocking Strategy:** 외부 의존성인 `src.common.log.LogManager`와 `request_id_ctx`는 `unittest.mock`을 사용하여 철저히 격리(Isolation)한다.
- **Verification Method:** 로그 호출 여부(Call Count), 로그 내용(Arguments), 반환값(Return Value), 예외 발생(Exception Raise) 여부를 검증한다.
