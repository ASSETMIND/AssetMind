## 1. 개요 (Overview)

본 문서는 AssetMind 회원가입 프로세스의 순수 비즈니스 로직을 담당하는 커스텀 훅 `useSignupLogic`의 동작을 검증하기 위한 단위 테스트(Unit Test) 명세이다.
UI 컴포넌트와 분리된 상태 관리, 입력 유효성 검증, 그리고 단계별 API 호출 제어(State Machine)가 외부 의존성 없이 올바르게 수행되는지 중점적으로 확인한다.

### 1.1. 테스트 환경

- **Framework:** Jest, `@testing-library/react` (Hooks Testing)
- **Mocking Strategy:** `Infrastructure Layer` 완전 격리 (API Query/Mutation Hooks 전체 Mocking)
- **Key Verification:**
  - **Flow Control:** [중복 확인] → [인증 발송] → [인증 확인] → [가입] 순서 제어 검증
  - **State Integrity:** 입력값 변경에 따른 인증 상태 초기화(Reset) 및 동기화 확인
  - **Error Handling:** API 실패, 네트워크 오류, 엣지 케이스(Edge Case)에 대한 방어 로직 확인
