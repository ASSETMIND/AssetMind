## 1. 개요 및 목적
본 문서는 **AssetMind** 프로젝트 팀원들의 기여도를 공정하고 정량적으로 산출하기 위한 기준을 정의한다.
단순 작업량이 아닌 **'기술적 난이도(Complexity)'**, **'시장 희소성(Scarcity)'**, **'프로젝트 영향도(Impact)'**를 반영하여, 포트폴리오 활용 시 객관적인 지표로 활용하는 것을 목적으로 한다.

---

## 2. 기여도 산출 공식
모든 개별 태스크는 **기본 점수(Fibonacci)**와 **직군 가중치(Role Multiplier)**를 곱하여 산정한다.

$$\text{최종 점수} = \sum (\text{태스크 기본 점수} \times \text{직군 가중치})$$

---

## 3. 태스크 등급 및 기본 점수표 (Task Levels)
모든 활동(개발, 문서작성, 핫픽스, 유지보수)은 피보나치 수열(1, 3, 5, 8, 13)에 기반한 5단계 등급으로 분류한다.

| 레벨 | 점수 | 등급명 (Definition) | 정의 및 기준 | 산출물 및 활동 예시 | 핫픽스(Hotfix) 및 유지보수 |
| :---: | :---: | :--- | :--- | :--- | :--- |
| **L5** | **13** | **Critical**<br>(핵심/설계) | • 대체 불가능한 최상위 난이도<br>• 시스템 근간/아키텍처 설계<br>• R&D 및 기술 검증 필수 | • ADRs (아키텍처 의사결정서)<br>• Kafka/Airflow 클러스터 구축<br>• ML 모델 아키텍처 설계<br>• OAuth2/JWT 보안 아키텍처 | **[Emergency]**<br>• 서비스 전체 중단(Down) 해결<br>• 데이터 유실/보안 사고 긴급 패치<br>• 새벽/휴일 긴급 대응 |
| **L4** | **8** | **Complex**<br>(고난이도) | • 복잡한 로직 및 의존성 해결<br>• 성능 최적화 필수<br>• 실패 시 핵심 기능 마비 | • API 명세서 / DB 설계서 (ERD)<br>• 복잡한 비즈니스 로직(정산/결제)<br>• CI/CD 파이프라인 구축<br>• 전역 상태 관리 설계 | **[Major Bug]**<br>• 핵심 기능(로그인, 결제) 오류 수정<br>• 잘못된 데이터 적재 수정<br>• 성능 저하(Time-out) 해결 |
| **L3** | **5** | **Standard**<br>(표준) | • 일반적인 기능 구현<br>• 기술 조사(Spike)<br>• 기술 부채 상환 | • 프로덕트/스프린트 명세서<br>• 일반 CRUD API 개발<br>• UI 컴포넌트 개발<br>• 리팩토링(코드 정리) | **[Normal Bug]**<br>• 특정 페이지 접근 불가 수정<br>• API 응답 오류 수정<br>• UI 깨짐 수정 (기능 영향 O) |
| **L2** | **3** | **Simple**<br>(단순) | • 정형화된 구현 패턴<br>• 단순 반복 작업 | • 유저 스토리 / 유즈 케이스<br>• 테스트 명세서 / 리뷰 명세서<br>• 단순 퍼블리싱 (HTML/CSS)<br>• DTO/VO 생성 | **[Minor Bug]**<br>• 오타 수정<br>• 단순 UI/UX 불편 사항 개선<br>• 로그 레벨 변경 |
| **L1** | **1** | **Minor**<br>(기타) | • 설정 변경 및 단순 관리<br>• 회의록 정리 | • 회의록 작성<br>• 라이브러리 버전 업데이트<br>• 환경 설정 값(Config) 변경 | **[Trivial]**<br>• 주석 추가/삭제<br>• 문서 현행화<br>• 변수명 변경 |

## 4. 직군별 가중치 (Role Multipliers)
기술적 진입 장벽과 시장 평균 연봉 데이터를 기반으로 직군별 가중치를 적용한다. 이는 동일 시간 투입 대비 기술적 기여도의 차이를 보정하기 위함이다.

| 그룹 | 세부 직군 (Role) | 가중치 | 주요 담당 업무 및 근거 |
| :--- | :--- | :---: | :--- |
| **AI / Data** | **머신러닝 개발자**<br>(Tensorflow) | **x 1.6** | • 모델 학습/튜닝 및 추론 최적화<br>• R&D 성격이 강하며 시장 희소성 최상 |
| **Infra** | **데이터 인프라 관리자**<br>(Airflow, MLFlow) | **x 1.4** | • 데이터 파이프라인 오케스트레이션<br>• 엔지니어링 능력과 데이터 이해도 동시 요구 |
| **Infra** | **인프라 관리자**<br>(Docker, ELK) | **x 1.35** | • 시스템 안정성 및 CI/CD 책임 (DevOps)<br>• 전체 인프라 통제권 보유 |
| **Backend** | **데이터 관리자**<br>(Kafka, Redis, DB) | **x 1.3** | • 대용량 트래픽 처리 및 트랜잭션 관리<br>• 분산 시스템 운영의 핵심 |
| **Backend** | **서버 개발자**<br>(Spring, JPA, Security) | **x 1.2** | • 핵심 비즈니스 로직 구현<br>• 인증/인가 및 API 서버 구축 |
| **Data** | **데이터 엔지니어**<br>(Python, Pandas) | **x 1.15** | • 데이터 전처리 및 ETL 스크립트 작성<br>• 분석과 엔지니어링의 중간 단계 |
| **Frontend** | **웹 개발자**<br>(React) | **x 1.1** | • 클라이언트 로직 구현 및 시각화<br>• UI/UX 인터랙션 개발 |
| **PM** | **기획자** | **x 1.05** | • 요구사항 정의 및 일정 관리<br>• 커뮤니케이션 비용 절감 기여 |
| **Design** | **UI 디자이너** | **x 1.0** | • 프로토타이핑 및 디자인 리소스 제작<br>• 시각적 가이드라인 수립 |

## 5. 적용 예시 (Simulation)

**상황 가정:** 릴리즈 마감 직전 긴급 수정 및 일반 수정 작업 수행 시

1.  **팀원 A (서버 개발자, x1.2)**
    * **작업:** 로그인 보안 버그 긴급 패치 (Emergency Hotfix)
    * **등급:** L5 (13점)
    * **계산:** $13 \times 1.2 = \mathbf{15.6 \text{점}}$

2.  **팀원 B (웹 개발자, x1.1)**
    * **작업:** 회원가입 화면 오타 수정 (Minor Hotfix)
    * **등급:** L1 (1점)
    * **계산:** $1 \times 1.1 = \mathbf{1.1 \text{점}}$

> **결과 해석:** 동일한 '버그 수정' 행위라도 기술적 난이도와 직군 가중치에 따라 점수 차이가 발생하며, 이는 프로젝트 안정성에 기여한 실질적 가치를 나타낸다.
