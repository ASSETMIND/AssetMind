# 결정사항

## 1. 프로젝트 의사결정 과정

프로젝트를 수행하면서 다음과 같은 순서대로 의사결정을 수행한다. 해당 방식은 빠른 논의가 가능하지만, 높은 레벨의 의사결정자의 많은 시간이 투자된다는 점과 계획의 유동성을 확보하기 힘들기 때문에 추후 변경되도록 한다.

1. [PM > UI] 간단한 유저스토리 명세 작성 후 전달
2. [UI > UX] 페이지 디자인 명세 작성 후 전달
3. [UX > BE] 유즈케이스 명세 작성 후 전달
4. [BE] API 명세 작성
5. [SPRINT MEETING] 스프린트 회의 전 모든 명세를 숙지하고, 15분 내의 빠른 회의 진행. 스프린트 계획서 작성
6. [DEVELOP] 스프린트 계획서대로 개발 진행
7. [TEST/OPT] 스프린트 계획서대로 테스트 및 최적화 진행
8. [REVIEW] 코드 주석 처리 및 리팩토링 진행. 본인 작업 사항을 기반으로 기여도 평가 진행

## 4. 프로젝트 디렉토리 구조

프로젝트 디렉토리 구조는 다음과 같이 확정함

- `apps/web` : [Front] React 웹 어플리케이션
- `apps/[MAS server]` : [MAS Server] 각 마이크로 서버 구축
- `libs/java-common` : [Shared Layer] 서비스 간 공통 코드
- `libs/web-ui` : [Front] 공통 디자인 시스템 컴포넌트
